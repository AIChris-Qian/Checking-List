**1. `[ 互联网常用指标 ]`**    
------------------------------

#### 1.1 用户数据：
   
   用户来源：`渠道`
   
   用户存量：`DAU / MAU`
   
   用户增量：`新增用户`
   
   用户健康度：`留存率` 
    

#### 1.2 行为数据：
    
   访问次数/频率：`PV / UV`
    
   访问时长：`时长`
   
   访问转化：`转化率`
   
   访问跳出：`弹出率`
    
    
#### 1.3 业务数据：

   总量：`GMV`
    
   人均：`ARPU / ARPPU / 人均访问时长`
   
   人数：`付费人数 / 播放人数`
   
   健康程度：`付费率 / 播放率`

**2. `[ 产品上线注意的指标 ]`**    
------------------------------

   AARRRR：`Acquisition（获取） / Activation（激活） / Retention（留存） / Revenue（收入） / Referral（自传播） / Recall（召回）`

#### 2.1 获取用户：

   `浏览量 / 点击量 / 下载量 / DNU (日新增用户) / CAC (用户获取成本) / ROI (投资回报率)`

#### 2.2 提高活跃度：

   `DAU (日活跃用户) / MAU（月活跃用户） / 在线使用时长 / 在线使用次数 / PV / UV`

#### 2.3 提高留存率：

   `次日留存率 / 7日留存率 / 30日留存率`  

#### 2.4 获取收入：

   `GMV（成交总额) / ARPU (平均每用户收入) / ARPPU (平均每付费用户收入) / LTV (生命周期价值) / 付费用户数 / 付费用户率` 

#### 2.5 用户推荐：

   `邀请发送量 / 病毒传播周期 / K因子 (平均每个用户带来的新用户)` 

#### 2.6 用户召回：

   `流失用户数 / 召回数`

**3. `[ 搭建指标体系 ]`**    
------------------------------

设计指标体系：`需求来源 / 确定一级指标 / 得到二级指标 / 得到三级指标`

落地指标体系：`数据采集 / 数据应用`

#### `[ 设计指标体系 ]`


#### 3.1 需求来源：
   
   先有**目标方案**，再有**数据指标**

   初期：`战略驱动`  全方位指标监控
   
   中期：`业务驱动`  搭建指标衡量现有业务
   
   后期：`数据驱动`  查缺补漏，通过数据来反向推动产品的迭代优化
   

#### 3.2 确定一级指标：
   
   一级指标：`产品在各个重要方面的运营情况`
   
   用户的生命周期：`AARRRR`

    获取用户 / 提高活跃度 / 提高留存率 /  获取收入 / 用户推荐 / 用户召回
    

#### 3.3 得到二级指标：
   
   二级指标：`通过对一级指标的成分拆解从而得到二级指标`  
   
    
#### 3.4 得到三级指标：
   
   三级指标：`通过对二级指标的路径拆解从而得到三级指标`  
   
   
#### `[ 落地指标体系 ]`

#### 3.5 数据采集：
   
   `行为主体 / 时间 / 主体描述 / 地理位置 / 时间描述`
   
   who (行为主体)
     
     用户ID：用户登录
     设备ID：用户未登录

   When (时间)
   
     UNIX时间：事件发生的时间
     
   What (主体描述)
   
     用户主体：性别 / 年龄 / 职业
     应用主体：应用版本
     设备主体：设备型号
     
  Where (地理位置)
  
     GPS定位 / IP定位
     
  How (事件描述)
     
     事件来源：多个行为造成同一个结果 / 一个行为造成不同结果 
     事件结果：该事件的具体信息描述
     
#### 3.6 数据应用：

    KPI分析：目标进度一目了然    实时分析：实时监控核心指标     事件分析：追踪或记录的用户行为        
    漏斗分析：监控转化率         留存分析：验证用户粘性         用户分群：划分用户群
