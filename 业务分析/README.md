**2. `[ 产品上线注意的指标 ]`**    
------------------------------

   AARRRR：`Acquisition（获取） / Activation（激活） / Retention（留存） / Revenue（收入） / Referral（自传播） / Recall（召回）`

#### 2.1 获取用户：

   `浏览量 / 点击量 / 下载量 / DNU (日新增用户) / CAC (用户获取成本) / ROI (投资回报率)`

#### 2.2 提高活跃度：

   `DAU (日活跃用户) / MAU（月活跃用户） / 在线使用时长 / 在线使用次数 / PV / UV`

#### 2.3 提高留存率：

   `次日留存率 / 7日留存率 / 30日留存率`  

#### 2.4 获取收入：

   `GMV（成交总额) / ARPU (平均每用户收入) / ARPPU (平均每付费用户收入) / LTV (生命周期价值) / 付费用户数 / 付费用户率` 

#### 2.5 用户推荐：

   `邀请发送量 / 病毒传播周期 / K因子 (平均每个用户带来的新用户)` 

#### 2.6 用户召回：

   `流失用户数 / 召回数`

**5. `[ 指标拆解 ]`**    
------------------------------

DAU (日活跃用户数量) 
     
     日新增用户 + 留存用户 + 回流用户

GMV (总消费额) 
  
     用户数 * 购买频次 * 客单价 = 用户数 * 付费率 * 客单价
                                     
LTV (生命周期总价值) 
    
     LT（生命周期）* ARPU（每个用户平均花费）
     sum（日活跃 * 日消费）/ 总用户 
     总留存率 * ARPU（假设 ARPU 是常数）

ROI (投资回报率)
   
    利润总额 / 投入成本总额

CAC (单个用户的获取成本) 
   
    市场总花费 / 新增用户数

病毒因子

    邀请率：邀请数 / 现有用户数
    接受率：进群人数 / 邀请数
    
    邀请率 * 接受率

**6. `[ 分析指标变动及原因 ]`**    
------------------------------

`上升/下降类   数据异常类`

1.	验证数据准确性 
2.  横向分析，定位指标变动的维度

              新用户：用户渠道来源
              老用户：用户生命周期
              用户画像：用户基本属性 (年龄 / 性别 / 地域)   
                       用户行为 (浏览 / 下单 / 观看)
              产品版本         

3.	纵向分析，确定流程路径进行指标拆解
4.	内部原因：

           获客 (渠道质量低，活动获取非目标用户) 
           需求 (新功能改动引起某些用户不满)
           提活手段 (签到等手段没达到目标，产品自然使用周期低导致大量用户短期内不再使用)
           
    外部原因：
    
           Pest模型
           政治 (政策影响)
           经济 (短期内主要是竞争环境，如对竞争对手的活动)
           社会 (舆论压力，用户生活方式变化，消费心理变化，价值观变化)
           技术 (创新解决方案的出现，分销渠道变化)

**7. `[ AB测试 ]`**    
------------------------------

#### 7.1 简述AB测试：

   流程：`试验分组 / 进行试验 / 分析结果`
     
    在 App / PC端 同时设计两个版本，在同一时间维度下，让组成成份相似的访客随机访问这些版本，收集用户体验数据和业务数据，评估出最好的版本
    
#### 7.2 常用的AB测试分组方法：

    基于设备号 / 用户ID尾号进行分组 (奇数 / 偶数)分为两组
   
   试验并行：`同一批次用户作为多个试验的数据源`
   
   
#### 7.3 面对多个试验并行的情况，保证分组的合理性：

   独占域：在试验期间，不会对这些用户进行其他试验
   
   共享域：用户同时进行多组AB测试
          
          基于正交法，上层的各个分组的用户随机选取 50% 进入下层的分组中，保证下层用户的随机性。
          
#### 7.4 证明AB测试分组的随机性：
     
   基于正交的方法可以保证用户分组的随机性
   防止意外，引入AA测试。
   
      AB版本分别按照6：4比例划分，从A版本中划分20%进行AA测试。
      验证结果时，要先保证AA测试通过，确保分组的合理性，然后再看AB测试是否通过。
     
#### 7.5 简述AB测试背后的理论支撑：

   中心极限定理：`样本足够大时，样本均值服从正态分布`
   
   结合假设检验，推翻或接受原假设。
   
     假设检验 / 置信区间 / 置信度 / P值 / x^2检验 / t检验 / F检验
   
#### 7.6 AB测试流程：

1.  收集指标，建立指标体系
       
        KPI指标 / 用户行为指标 / 试验提升指标：转化率
   
2.	分配流量，AA实验 

        抽取一部分流量，选择任意两组不加策略空跑，监控核心指标数据，选取两组数据最接近的上实验。

3.  假设检验   
    
        (1) 确定实验周期
        (2) 确定第一类错误 α 和第二类错误 β，估计样本的标准差，以此来确定实验所需样本量
        (3) 确定两个总体参数检验的统计量和分布
                                            H0：转化率没有差异 
                                            H1：转化率有差异
        (4) 计算统计量是否在显著性水平内，如果在显著性水平内则拒绝原假设，否则没有充分证据拒绝原假设

4.  决策方案
       
        (1) 正收益：确定发布新版本           改进在显著性水平内，收益提升率达到预期水平，证明了 [ 转化率提升] 
        (2) 负收益：优化迭代方案重新开发      改进在显著性水平内，收益提升率没有达到预期水平，没有证明 [ 转化率提升 ]
        (3) 持平：调整分流比例继续测试       改进不在显著性水平内，无法证明 [ 转化率提升的假设 ]

**8. `[ 用户运营体系 ]`**    
------------------------------    

#### 8.1 用户分层：

    基于大方向的划分，希望用户朝什么核心目标努力。
    用户分层可以是单层金字塔结构，也可以是双层金字塔结构。中心思想：根据指标划分
    
   `新用户 / 下载用户 / 活跃用户 / 兴趣用户 / 付费用户` 
   
1.  注册：新用户
       
        希望新用户能下载产品，可以考虑新用户福利。
   
2.  下载：下载用户
   
        希望下载用户能使用产品，应该使用新手引导尽快熟悉。
      
3.  使用：活跃用户
      
        希望活跃用户加深使用产品的频率，可以考虑固化用户的使用习惯。
      
4.  下单：兴趣用户
      
        希望兴趣用户完成付费决策，购买商品，可以考虑促销手段。
      
5.  购买：付费用户
   
        这是目标用户，希望付费用户能一直维持这状态。
      

#### 8.2 用户分群：

    将同一个分层内的群体继续切分，满足更高的精细化需要    
    RFM模型可以衡量消费用户的价值和创利能力，是一个典型的分群
    
  `一：建立指标，以指标作为划分依据     二：使用K均值聚类算法，通过数据挖掘建立用户分群`
  
  RFM模型:
  
    三个核心指标：最近一次消费时间 / 消费频率 / 消费金额
    
    最近一次消费时间 Recency：衡量用户的流失，消费时间越近，越容易维系关系。
    消费频率 Frequency：用户在限定的期间内购买的次数，用户购买的次数越多，忠诚度也越高。
    消费金额 Monetary：直接反应用户的对企业利润的贡献。
    
    重要价值客户：R↑ F↑ M↑   重要发展客户：R↑ F↓ M↑     重要挽留客户：R↓ F↓ M↑    重要保持客户：R↓ F↑ M↑   

1.  建立指标，以指标作为划分依据
     
        指标：用描述性统计的分位数 (中位数 / 四分位数)
        
2.  使用K均值聚类算法，通过数据挖掘建立用户分群

        RFM三个指标作为特征输入，零均值归一化(z-score)处理，清洗异常极值，建立可视化的散点图。
        K均值聚类算法以距离作为目标函数，在距离上越接近的两个用户，其相似的可能性也越大。

**9. `[ 归因分析 ]`**    
------------------------------    

    针对当前的场景和目标，将用户点击行为 [ 贡献 ] 合理分配到每一个渠道。

#### 9.1 归因分析模型参数：

   目标转化事件：`支付订单 / 提交订单详情`
   
   前向关联事件：`浏览商品详情 / 商品加入购物车`
   
   待归因事件：`广告位的点击 / 浏览`

#### 9.2 归因分析模型：

1.  首次归因：新品牌、新产品推广
   
        多个 [ 待归因事件 ] 对同一个 [ 目标转化事件 ] 做出贡献时，认为第一个 [ 待归因事件 ] 贡献为100%
   
2.  末次归因：吸引客户购买
   
        多个 [ 待归因事件 ] 对同一个 [ 目标转化事件 ] 做出贡献时，认为最后一个 [ 待归因事件 ] 贡献为100%
   
3.  线性归因：在整个销售周期内保持与客户的联系
    
        多个 [ 待归因事件 ] 对同一个 [ 目标转化事件 ] 做出贡献时，认为每一个 [ 待归因事件 ] 平均分配贡献
   
4.  位置归因：重视线索来源和促成销售渠
   
        多个 [ 待归因事件 ] 对同一个 [ 目标转化事件 ] 做出贡献时，认为第一个和最后一个 [ 待归因事件 ] 各贡献40%，其余 [ 待归因事件 ] 平均分配剩余20%

5.  时间衰减归因：做短期的促销
   
        多个 [ 待归因事件 ] 对同一个 [ 目标转化事件 ] 做出贡献时，越靠近 [ 待归因事件 ] 做出的贡献越大

**10. `[ 费米问题 ]`**    
------------------------------    

    估算类问题：从供需入手，考虑供给端和需求端
    
1.  煎饼摊子一年能卖多少个煎饼？
   
        每天制作的煎饼数 * 365 
        
        一个小时能制作的煎饼数 * 每天出售的时间段 (早上 + 晚上) * 365
   
2.  北京有多少个加油站？
   
        北京每天需要加油的车辆数 / 每天每个加油站能加几辆车 
        
        (总车数 / 加油间隔天数) / (加油站的工作时长 * 每小时能加的车辆)


3.  南京外卖骑手共有多少人？
   
        吃饭高峰的点单总数 / 一个外卖骑手高峰时间段能送多少份
        
        (高峰期18-30岁点单的人数 + 高峰期30-45岁点单的人数) / (高峰期时长 * 一小时送外卖的趟数 * 每趟送几份外卖)


4.  北京有多少辆出租车？
   
        北京每天乘坐出租车的人数 / 辆出租车每天的平均载客数
        
        (北京的总人口数 × 北京每天乘坐出租车的人口比例) / (每天工作时长 * 一小时载客的趟数 * 每趟载客量)
