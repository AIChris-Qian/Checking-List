#### 1.1 用户数据：

   用户存量：`DAU / MAU`

   访问次数/频率：`PV / UV`

   访问跳出：`弹出率`
    
    
#### 1.3 业务数据：

   总量：`GMV`
    
   人均：`ARPU / ARPPU / 人均访问时长`

   健康程度：`付费率 / 播放率`

**2. `[ 产品上线注意的指标 ]`**    
------------------------------

   AARRRR：`Acquisition（获取） / Activation（激活） / Retention（留存） / Revenue（收入） / Referral（自传播） / Recall（召回）`

#### 2.1 获取用户：

   `浏览量 / 点击量 / 下载量 / DNU (日新增用户) / CAC (用户获取成本) / ROI (投资回报率)`

#### 2.2 提高活跃度：

   `DAU (日活跃用户) / MAU（月活跃用户） / 在线使用时长 / 在线使用次数 / PV / UV`

#### 2.3 提高留存率：

   `次日留存率 / 7日留存率 / 30日留存率`  

#### 2.4 获取收入：

   `GMV（成交总额) / ARPU (平均每用户收入) / ARPPU (平均每付费用户收入) / LTV (生命周期价值) / 付费用户数 / 付费用户率` 

#### 2.5 用户推荐：

   `邀请发送量 / 病毒传播周期 / K因子 (平均每个用户带来的新用户)` 

#### 2.6 用户召回：

   `流失用户数 / 召回数`

#### `[ 设计指标体系 ]`


#### 3.1 需求来源：
   
   先有**目标方案**，再有**数据指标**

   初期：`战略驱动`  全方位指标监控
   
   中期：`业务驱动`  搭建指标衡量现有业务
   
   后期：`数据驱动`  查缺补漏，通过数据来反向推动产品的迭代优化
   

#### 3.2 确定一级指标：
   
   一级指标：`产品在各个重要方面的运营情况`
   
   用户的生命周期：`AARRRR`

    获取用户 / 提高活跃度 / 提高留存率 /  获取收入 / 用户推荐 / 用户召回
    

#### 3.3 得到二级指标：
   
   二级指标：`通过对一级指标的成分拆解从而得到二级指标`  
   
    
#### 3.4 得到三级指标：
   
   三级指标：`通过对二级指标的路径拆解从而得到三级指标`  
   
   
#### `[ 落地指标体系 ]`

#### 3.5 数据采集：
   
   `行为主体 / 时间 / 主体描述 / 地理位置 / 时间描述`
   
   who (行为主体)
     
     用户ID：用户登录
     设备ID：用户未登录

   When (时间)
   
     UNIX时间：事件发生的时间
     
   What (主体描述)
   
     用户主体：性别 / 年龄 / 职业
     应用主体：应用版本
     设备主体：设备型号
     
  Where (地理位置)
  
     GPS定位 / IP定位
     
  How (事件描述)
     
     事件来源：多个行为造成同一个结果 / 一个行为造成不同结果 
     事件结果：该事件的具体信息描述
     
#### 3.6 数据应用：

    KPI分析：目标进度一目了然    实时分析：实时监控核心指标     事件分析：追踪或记录的用户行为        
    漏斗分析：监控转化率         留存分析：验证用户粘性         用户分群：划分用户群

**5. `[ 指标拆解 ]`**    
------------------------------

DAU (日活跃用户数量) = 日新增用户 + 留存用户 + 回流用户

GMV (总消费额) = 用户数 * 购买频次 * 客单价 = 用户数 * 付费率 * 客单价
                                     
LTV (生命周期总价值) = LT（生命周期）* ARPU（每个用户平均花费）= sum（日活跃 * 日消费）/ 总用户 = 总留存率 * ARPU（假设 ARPU 是常数）

ROI (投资回报率) = 利润总额 / 投入成本总额

CAC (单个用户的获取成本) =市场总花费 / 新增用户数

**6. `[ 分析指标变动及原因]`**    
------------------------------

`上升/下降类   数据异常类`

1.	验证数据准确性 
2.  横向分析，定位指标变动的维度

              新用户：用户渠道来源
              老用户：用户生命周期
              用户画像：用户基本属性 (年龄 / 性别 / 地域)   
                       用户行为 (浏览 / 下单 / 观看)
              产品版本         

3.	纵向分析，确定流程路径进行指标拆解
4.	内部原因：

           获客 (渠道质量低，活动获取非目标用户) 
           需求 (新功能改动引起某些用户不满)
           提活手段 (签到等手段没达到目标，产品自然使用周期低导致大量用户短期内不再使用)
           
    外部原因：
    
           Pest模型
           政治 (政策影响)
           经济 (短期内主要是竞争环境，如对竞争对手的活动)
           社会 (舆论压力，用户生活方式变化，消费心理变化，价值观变化)
           技术 (创新解决方案的出现，分销渠道变化)
